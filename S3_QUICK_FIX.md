# Быстрое решение проблем с S3 файлами

## Проверка файлов

Запустите скрипт проверки:
```bash
npm run check:s3
```

Скрипт покажет:
- ✅ Какие файлы доступны
- ❌ Какие файлы повреждены
- 📊 Статистику по всем файлам

## Частые причины проблем

### 1. Неправильный Content-Type
**Симптом:** Браузер не показывает изображение  
**Решение:** Перезагрузите файл через админку

### 2. Проблемы с доступом (403 Forbidden)
**Симптом:** Ошибка 403 при открытии файла  
**Решение:** 
1. Зайдите в Timeweb панель
2. Откройте настройки bucket
3. Включите "Публичный доступ"
4. Добавьте CORS правила

### 3. Файл не найден (404)
**Симптом:** Ошибка 404  
**Решение:** Файл был удален, загрузите заново

### 4. CORS ошибка
**Симптом:** В консоли браузера "CORS policy"  
**Решение:** Настройте CORS в Timeweb:

```json
{
  "CORSRules": [
    {
      "AllowedOrigins": ["*"],
      "AllowedMethods": ["GET", "PUT", "POST"],
      "AllowedHeaders": ["*"]
    }
  ]
}
```

## Быстрое исправление

### Вариант 1: Перезагрузка файла
1. Откройте админку → Фильмы
2. Найдите фильм с проблемным файлом
3. Удалите старый файл
4. Загрузите новый

### Вариант 2: Проверка URL
Откройте URL файла в браузере:
```
https://your-s3-url.com/bucket/file.jpg
```

Если не открывается:
- 403 → Проблема с доступом
- 404 → Файл не существует
- Загружается вместо показа → Неправильный Content-Type

## Настройка Timeweb S3

### Шаг 1: Публичный доступ
1. Timeweb → S3 → Ваш bucket
2. Настройки → ACL
3. Включите "Публичное чтение"

### Шаг 2: CORS
1. Настройки → CORS
2. Добавьте правило:
   - Origins: `*`
   - Methods: `GET, PUT, POST`
   - Headers: `*`

### Шаг 3: Bucket Policy
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::your-bucket/*"
    }
  ]
}
```

## Проверка конкретного файла

В консоли браузера:
```javascript
// Проверка доступности
fetch('https://your-s3-url.com/bucket/file.jpg')
  .then(r => console.log('Status:', r.status, 'OK:', r.ok))
  .catch(e => console.error('Error:', e));

// Проверка заголовков
fetch('https://your-s3-url.com/bucket/file.jpg')
  .then(r => {
    console.log('Content-Type:', r.headers.get('content-type'));
    console.log('Content-Length:', r.headers.get('content-length'));
  });
```

## Превентивные меры

1. ✅ Всегда проверяйте файл после загрузки
2. ✅ Используйте правильные типы файлов (JPEG, PNG, MP4)
3. ✅ Не загружайте файлы > 200MB без chunked upload
4. ✅ Регулярно запускайте `npm run check:s3`

## Если ничего не помогает

1. Проверьте переменные окружения в `.env`:
   ```
   S3_URL=https://...
   S3_ACCESS_KEY=...
   S3_SECRET_ACCESS_KEY=...
   BUCKET_NAME=...
   ```

2. Проверьте что bucket существует в Timeweb

3. Проверьте что ключи доступа действительны

4. Попробуйте загрузить файл вручную через Timeweb панель

5. Свяжитесь с поддержкой Timeweb
