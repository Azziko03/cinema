# Реализация админ-панели с 2FA через Telegram

## Что реализовано

### ✅ Двухфакторная аутентификация через Telegram

1. **Первый фактор**: Email + пароль
   - Проверка в базе данных
   - Проверка роли `admin`
   - Проверка статуса аккаунта

2. **Второй фактор**: 6-значный код через Telegram
   - Генерируется случайный код
   - Отправляется через Telegram Bot API
   - Действителен 5 минут
   - Одноразовый (удаляется после использования)

### ✅ Разделение доступа

**Администраторы** (`role: admin`):
- ✅ Доступ только к `/admin/*` маршрутам
- ✅ НЕ могут заходить на страницы пользователей
- ✅ Вход через `/admin/login` с 2FA
- ✅ Отдельная сессия через NextAuth

**Обычные пользователи** (`role: user`):
- ✅ Доступ к пользовательским страницам
- ✅ НЕ могут заходить на админские страницы
- ✅ Вход через `/auth/signin`

### ✅ Защита маршрутов (Middleware)

Middleware автоматически:
- ✅ Перенаправляет админов с пользовательских страниц на `/admin/dashboard`
- ✅ Перенаправляет пользователей с админских страниц на `/`
- ✅ Требует авторизацию для защищенных маршрутов
- ✅ Проверяет роль пользователя

### ✅ Интерфейс

**Страница входа** (`/admin/login`):
- ✅ Красивый дизайн с градиентом
- ✅ Два шага: логин → код
- ✅ Переключатель языка (RU/KG)
- ✅ Обработка ошибок
- ✅ Индикаторы загрузки

**Дашборд** (`/admin/dashboard`):
- ✅ Статистика (пользователи, фильмы, заказы, выручка)
- ✅ Быстрые действия
- ✅ Кнопка выхода
- ✅ Приветствие администратора

### ✅ API Endpoints

1. **POST `/api/admin/login`**
   - Проверка email и пароля
   - Генерация и отправка кода в Telegram
   - Возвращает `userId` для второго шага

2. **POST `/api/admin/verify-code`**
   - Проверка 6-значного кода
   - Проверка срока действия
   - Возвращает данные пользователя

3. **GET `/api/translations`**
   - Загрузка переводов для клиентских компонентов
   - Поддержка RU и KG языков

### ✅ Переводы

Все тексты переведены на:
- 🇷🇺 Русский (`app/i18n/locales/ru/auth.json`)
- 🇰🇬 Кыргызский (`app/i18n/locales/kg/auth.json`)

Секция `admin` включает:
- Заголовки и подзаголовки
- Поля формы
- Кнопки
- Сообщения об ошибках
- Информационные сообщения

### ✅ База данных

**Таблица `settings`**:
- `ADMIN_TELEGRAM_BOT_TOKEN` - токен бота
- `ADMIN_TELEGRAM_USER_ID` - Telegram ID администратора

**Таблица `users`**:
- `verificationCode` - временный код 2FA
- `codeExpiresAt` - время истечения кода

### ✅ Безопасность

1. **Хранение настроек**: Telegram токены в БД, не в .env
2. **Коды 2FA**: Одноразовые, с ограниченным сроком действия
3. **Разделение ролей**: Строгая изоляция админов и пользователей
4. **Middleware**: Автоматическая защита всех маршрутов
5. **Валидация**: Zod схемы для всех API endpoints
6. **Хеширование паролей**: bcrypt с 10 раундами

## Как это работает

### Процесс входа

```
1. Пользователь открывает /admin/login
   ↓
2. Вводит email и пароль
   ↓
3. POST /api/admin/login
   ├─ Проверка email/пароля в БД
   ├─ Проверка роли admin
   ├─ Генерация 6-значного кода
   ├─ Сохранение кода в БД (5 мин)
   └─ Отправка кода в Telegram
   ↓
4. Пользователь получает код в Telegram
   ↓
5. Вводит код на странице
   ↓
6. POST /api/admin/verify-code
   ├─ Проверка кода
   ├─ Проверка срока действия
   └─ Удаление кода из БД
   ↓
7. signIn через NextAuth
   ↓
8. Redirect на /admin/dashboard
```

### Защита маршрутов

```
Middleware проверяет каждый запрос:

1. Публичные маршруты → разрешить
2. /admin/login без авторизации → разрешить
3. /admin/* без авторизации → redirect /admin/login
4. /admin/* с role=user → redirect /
5. /admin/* с role=admin → разрешить
6. /auth/* с role=admin → redirect /admin/dashboard
7. /profile с role=admin → redirect /admin/dashboard
8. /admin/* с role=user → redirect /
```
