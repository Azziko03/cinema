# Настройка админ-панели

## Обзор

Админ-панель защищена двухфакторной аутентификацией (2FA) через Telegram. При входе администратор вводит email и пароль, после чего на его Telegram отправляется 6-значный код подтверждения.

## Требования

1. Telegram аккаунт администратора
2. Telegram бот для отправки кодов
3. Пользователь с ролью `admin` в базе данных

## Шаг 1: Создание Telegram бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Сохраните токен бота (например: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

## Шаг 2: Получение Telegram User ID

1. Откройте Telegram и найдите [@userinfobot](https://t.me/userinfobot)
2. Отправьте любое сообщение боту
3. Бот ответит с вашим User ID (например: `123456789`)

## Шаг 3: Настройка базы данных

Добавьте настройки Telegram в таблицу `settings`:

```sql
-- Вставка токена бота
INSERT INTO settings (id, key, value)
VALUES (gen_random_uuid(), 'ADMIN_TELEGRAM_BOT_TOKEN', 'ВАШ_ТОКЕН_БОТА')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;

-- Вставка User ID администратора
INSERT INTO settings (id, key, value)
VALUES (gen_random_uuid(), 'ADMIN_TELEGRAM_USER_ID', 'ВАШ_TELEGRAM_USER_ID')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;
```

Или через Prisma Studio:

```bash
npx prisma studio
```

1. Откройте таблицу `Setting`
2. Добавьте две записи:
   - `key`: `ADMIN_TELEGRAM_BOT_TOKEN`, `value`: ваш токен бота
   - `key`: `ADMIN_TELEGRAM_USER_ID`, `value`: ваш Telegram User ID

## Шаг 4: Создание администратора

Если администратор еще не создан, выполните:

```bash
npm run seed
```

Это создаст администратора с данными:
- Email: `admin@cinema.kg`
- Пароль: `admin123`
- Роль: `admin`

**⚠️ ВАЖНО:** Измените пароль после первого входа!

Или создайте администратора вручную через SQL:

```sql
-- Хешируйте пароль с помощью bcrypt (rounds=10)
INSERT INTO users (id, email, password_hash, full_name, role, status, email_verified)
VALUES (
  gen_random_uuid(),
  'admin@example.com',
  '$2a$10$хешированный_пароль',
  'Администратор',
  'admin',
  'active',
  true
);
```

## Шаг 5: Проверка настройки

1. Откройте браузер и перейдите на `/admin/login`
2. Введите email и пароль администратора
3. Нажмите "Войти"
4. Проверьте Telegram - вы должны получить сообщение с 6-значным кодом
5. Введите код на странице подтверждения
6. После успешной проверки вы будете перенаправлены в админ-панель

## Архитектура безопасности

### Разделение доступа

- **Администраторы** (`role: admin`):
  - Доступ только к `/admin/*` маршрутам
  - НЕ могут заходить на страницы обычных пользователей (`/profile`, `/auth/signin`, и т.д.)
  - Вход через `/admin/login` с 2FA через Telegram
  
- **Обычные пользователи** (`role: user`):
  - Доступ к `/profile`, `/orders`, `/tickets` и другим пользовательским страницам
  - НЕ могут заходить на админские страницы (`/admin/*`)
  - Вход через `/auth/signin` (email/пароль или Google OAuth)

### Защита маршрутов

Middleware автоматически:
- Перенаправляет админов с пользовательских страниц на `/admin/dashboard`
- Перенаправляет пользователей с админских страниц на `/`
- Требует авторизацию для защищенных маршрутов
- Проверяет роль пользователя для доступа к админке

### Двухфакторная аутентификация

1. **Первый фактор**: Email + пароль
   - Проверка в базе данных
   - Проверка роли `admin`
   - Проверка статуса `active`

2. **Второй фактор**: Код из Telegram
   - Генерируется 6-значный код
   - Отправляется через Telegram Bot API
   - Действителен 5 минут
   - Одноразовый (удаляется после использования)

## API Endpoints

### POST `/api/admin/login`

Первый шаг входа - проверка email и пароля.

**Request:**
```json
{
  "email": "admin@cinema.kg",
  "password": "admin123"
}
```

**Response (успех):**
```json
{
  "message": "Код отправлен в Telegram",
  "userId": "uuid",
  "requiresTwoFactor": true
}
```

**Response (ошибка):**
```json
{
  "error": "Неверный email или пароль"
}
```

### POST `/api/admin/verify-code`

Второй шаг входа - проверка кода из Telegram.

**Request:**
```json
{
  "userId": "uuid",
  "code": "123456"
}
```

**Response (успех):**
```json
{
  "success": true,
  "message": "Код подтвержден",
  "user": {
    "id": "uuid",
    "email": "admin@cinema.kg",
    "name": "Администратор",
    "role": "admin"
  }
}
```

**Response (ошибка):**
```json
{
  "error": "Неверный или истекший код"
}
```

## Troubleshooting

### Код не приходит в Telegram

1. Проверьте, что бот создан и токен правильный
2. Убедитесь, что вы начали диалог с ботом (отправьте `/start`)
3. Проверьте настройки в таблице `settings`
4. Проверьте логи сервера на наличие ошибок

### Ошибка "Telegram settings not found"

Убедитесь, что в таблице `settings` есть записи:
- `ADMIN_TELEGRAM_BOT_TOKEN`
- `ADMIN_TELEGRAM_USER_ID`

### Ошибка "Доступ запрещен"

Убедитесь, что:
1. Пользователь существует в базе данных
2. Роль пользователя установлена как `admin`
3. Статус пользователя `active`

### Код истек

Коды действительны 5 минут. Нажмите "Отправить код повторно" для получения нового кода.

## Безопасность

### Рекомендации

1. **Храните токен бота в секрете** - не коммитьте его в Git
2. **Используйте сильные пароли** для администраторов
3. **Ограничьте количество попыток входа** (можно добавить rate limiting)
4. **Регулярно меняйте пароли** администраторов
5. **Мониторьте логи входа** в админ-панель
6. **Используйте HTTPS** в продакшене

### Дополнительные меры

Можно добавить:
- Rate limiting для API endpoints
- Логирование всех попыток входа
- IP whitelist для админ-панели
- Уведомления о входе в Telegram
- Сессии с ограниченным временем жизни
