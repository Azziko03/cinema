# Массовое создание сеансов

## Обзор

Реализована отдельная страница для массового создания сеансов с интеллектуальной проверкой доступности залов.

## Основные возможности

### 1. Отдельная страница добавления
- **URL:** `/admin/sessions/create`
- Убрана модалка, теперь полноценная страница
- Пошаговый интерфейс с 4 шагами

### 2. Множественный выбор параметров

#### Фильм (Шаг 1)
- Выбор одного фильма из списка
- Отображение постера, названия, длительности
- Возможность изменить статус фильма прямо при выборе

#### Залы (Шаг 2)
- **Множественный выбор залов** (checkbox)
- Можно выбрать несколько залов одновременно
- Отображение количества мест в каждом зале

#### Даты (Шаг 3)
- **Диапазон дат:** от startDate до endDate
- Если endDate не указан, создается только на один день
- Пример: с 15.05.2026 по 21.05.2026 = 7 дней

#### Время (Шаг 3)
- **Множественные временные слоты**
- Можно добавить несколько времен (например: 14:00, 17:00, 20:00)
- Кнопка "+ Добавить время" для добавления слотов
- Кнопка удаления для каждого слота

### 3. Интеллектуальная проверка доступности

#### Как работает:
1. Админ заполняет все параметры
2. Нажимает "Проверить доступность залов"
3. Система проверяет все комбинации:
   - Каждый день в диапазоне
   - Каждое время
   - Каждый зал
4. Показывает результаты:
   - Сколько слотов доступно
   - Сколько конфликтов
   - Список конфликтов с деталями

#### Проверка конфликтов:
```typescript
// Для каждой комбинации проверяется:
- Дата: 15.05.2026
- Время: 14:00
- Зал: Зал 1
- Длительность фильма: 120 минут
- Время окончания: 16:00

// Проверка пересечения с существующими сеансами:
if (
  (новый_старт >= существующий_старт && новый_старт < существующий_конец) ||
  (новый_конец > существующий_старт && новый_конец <= существующий_конец) ||
  (новый_старт <= существующий_старт && новый_конец >= существующий_конец)
) {
  // Конфликт!
}
```

### 4. Результаты проверки

#### Успешный результат:
```
✅ Результаты проверки
Доступно слотов: 18 из 21

⚠️ Конфликты расписания:
- Зал 1 - 16.05.2026 в 14:00
- Зал 2 - 18.05.2026 в 17:00
- Зал 1 - 20.05.2026 в 20:00
```

#### Если все занято:
```
⚠️ Нет доступных слотов для выбранных параметров
```

### 5. Массовое создание

После успешной проверки:
1. Появляется Шаг 4 с дополнительными параметрами:
   - Базовая цена
   - Язык (Оригинал/Дубляж)
   - Формат (2D/3D/IMAX)
2. Кнопка "Создать N сеансов" (где N = количество доступных слотов)
3. При нажатии создаются все доступные сеансы одновременно

## Примеры использования

### Пример 1: Один фильм на неделю
```
Фильм: Аватар 3
Залы: [Зал 1, Зал 2]
Даты: 15.05.2026 - 21.05.2026 (7 дней)
Время: [14:00, 17:00, 20:00]
Цена: 300 сом

Расчет:
7 дней × 3 времени × 2 зала = 42 слота
Если все свободны → создается 42 сеанса
```

### Пример 2: Один день, несколько сеансов
```
Фильм: Дюна 3
Залы: [Зал 1]
Даты: 15.05.2026 (только startDate)
Время: [10:00, 13:00, 16:00, 19:00, 22:00]
Цена: 350 сом

Расчет:
1 день × 5 времен × 1 зал = 5 слотов
Если все свободны → создается 5 сеансов
```

### Пример 3: Выходные дни
```
Фильм: Интерстеллар
Залы: [Зал 1, Зал 2, Зал 3]
Даты: 17.05.2026 - 18.05.2026 (суббота-воскресенье)
Время: [12:00, 15:00, 18:00, 21:00]
Цена: 400 сом

Расчет:
2 дня × 4 времени × 3 зала = 24 слота
```

## API Endpoints

### POST /api/admin/sessions/check-availability
Проверяет доступность залов для заданных параметров.

**Request:**
```json
{
  "movieId": "uuid",
  "hallIds": ["uuid1", "uuid2"],
  "startDate": "2026-05-15",
  "endDate": "2026-05-21",
  "timeSlots": ["14:00", "17:00", "20:00"],
  "durationMinutes": 120
}
```

**Response:**
```json
{
  "totalRequested": 42,
  "totalAvailable": 38,
  "totalConflicts": 4,
  "availableSlots": [
    {
      "hallId": "uuid",
      "hallName": "Зал 1",
      "date": "15.05.2026",
      "time": "14:00",
      "startTime": "2026-05-15T14:00:00.000Z",
      "endTime": "2026-05-15T16:00:00.000Z"
    },
    // ... остальные доступные слоты
  ],
  "conflicts": [
    {
      "hallId": "uuid",
      "hallName": "Зал 1",
      "date": "16.05.2026",
      "time": "14:00",
      "startTime": "2026-05-16T14:00:00.000Z",
      "endTime": "2026-05-16T16:00:00.000Z"
    },
    // ... конфликты
  ]
}
```

### POST /api/admin/sessions/bulk-create
Создает множество сеансов одновременно.

**Request:**
```json
{
  "movieId": "uuid",
  "availableSlots": [/* массив из check-availability */],
  "basePrice": 300,
  "language": "original",
  "format": "TWO_D"
}
```

**Response:**
```json
{
  "total": 38,
  "created": 38,
  "failed": 0,
  "errors": []
}
```

## Преимущества нового подхода

### 1. Экономия времени
**Было:** Создать 42 сеанса = 42 раза заполнить форму
**Стало:** Создать 42 сеанса = 1 раз заполнить форму

### 2. Предотвращение конфликтов
- Автоматическая проверка перед созданием
- Визуализация конфликтов
- Невозможно создать пересекающиеся сеансы

### 3. Гибкость
- Можно создать на один день или на неделю
- Можно выбрать один зал или все залы
- Можно добавить одно время или несколько

### 4. Прозрачность
- Видно сколько сеансов будет создано
- Видно какие слоты заняты
- Подтверждение перед созданием

## Workflow

```
1. Переход на /admin/sessions/create
         ↓
2. Выбор фильма (1 фильм)
         ↓
3. Выбор залов (можно несколько)
         ↓
4. Указание дат (диапазон или один день)
         ↓
5. Добавление времен (можно несколько)
         ↓
6. Нажатие "Проверить доступность"
         ↓
7. Просмотр результатов
         ↓
    ┌────┴────┐
    │         │
    ▼         ▼
Есть слоты  Нет слотов
    │         │
    │         └─→ Изменить параметры
    │
    ▼
8. Указание цены, языка, формата
    ↓
9. Нажатие "Создать N сеансов"
    ↓
10. Массовое создание в БД
    ↓
11. Редирект на /admin/sessions
```

## Технические детали

### Генерация дат
```typescript
const dates: Date[] = [];
const start = new Date(startDate);
const end = endDate ? new Date(endDate) : new Date(startDate);

for (let d = new Date(start); d <= end; d.setDate(d.getDate() + 1)) {
  dates.push(new Date(d));
}
```

### Проверка пересечений
```typescript
const hasConflict = existingSessions.some((existing) => {
  if (existing.hallId !== hallId) return false;

  const existingStart = new Date(existing.startTime);
  const existingEnd = new Date(existing.endTime);

  return (
    (startTime >= existingStart && startTime < existingEnd) ||
    (endTime > existingStart && endTime <= existingEnd) ||
    (startTime <= existingStart && endTime >= existingEnd)
  );
});
```

### Массовое создание
```typescript
for (const slot of availableSlots) {
  // Еще раз проверяем конфликты (защита от race conditions)
  const conflicts = await prisma.session.findMany({ /* ... */ });
  
  if (conflicts.length === 0) {
    await prisma.session.create({
      data: {
        movieId,
        hallId: slot.hallId,
        startTime: new Date(slot.startTime),
        endTime: new Date(slot.endTime),
        basePrice,
        language,
        format,
      },
    });
  }
}
```

## Безопасность

1. **Двойная проверка конфликтов:**
   - При check-availability
   - При bulk-create (защита от race conditions)

2. **Авторизация:**
   - Только админы могут создавать сеансы
   - Проверка на каждом API endpoint

3. **Валидация:**
   - Проверка существования фильма
   - Проверка существования залов
   - Валидация дат и времен

- ✅ Видеть конфликты до создания
- ✅ Экономить время администратора
