# Исправление проблемы со скачиванием трейлера

## Проблема
При клике на трейлер браузер начинал скачивать `.mov` файл вместо воспроизведения.

## Причина
Использовался `<iframe>` для всех типов видео, но iframe подходит только для YouTube. Для локальных видео файлов (из S3) нужен HTML5 `<video>` плеер.

## Решение

### 1. Определение типа трейлера
```typescript
function isYouTubeUrl(url: string): boolean {
  return url.includes('youtube.com') || url.includes('youtu.be');
}
```

### 2. Условный рендеринг
```tsx
{isYouTubeUrl(trailer.url) ? (
  // YouTube - используем iframe
  <iframe src={getYouTubeEmbedUrl(trailer.url)} />
) : (
  // Локальное видео - используем video tag
  <video controls autoPlay playsInline>
    <source src={trailer.url} type="video/mp4" />
    <source src={trailer.url} type="video/quicktime" />
    <source src={trailer.url} type="video/webm" />
  </video>
)}
```

## Поддерживаемые форматы

### YouTube трейлеры
- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/embed/VIDEO_ID`

→ Используется `<iframe>` с YouTube embed API

### Локальные трейлеры (S3)
- `.mp4` (рекомендуется)
- `.mov` (QuickTime)
- `.webm`

→ Используется HTML5 `<video>` плеер

## Особенности HTML5 video

### Атрибуты
- `controls` - показывает элементы управления (play, pause, volume, etc.)
- `autoPlay` - автоматическое воспроизведение
- `playsInline` - воспроизведение inline на iOS (без fullscreen)

### Множественные источники
```tsx
<source src={url} type="video/mp4" />
<source src={url} type="video/quicktime" />
<source src={url} type="video/webm" />
```
Браузер выберет первый поддерживаемый формат.

## Рекомендации

### Для YouTube трейлеров
1. Загрузите трейлер на YouTube
2. Скопируйте ссылку (любой формат)
3. Вставьте в поле "Трейлер" в админке

### Для локальных трейлеров
1. Конвертируйте видео в MP4 (H.264)
2. Оптимизируйте размер (рекомендуется < 50MB)
3. Загрузите через админку

**Рекомендуемые настройки для MP4:**
- Кодек: H.264
- Разрешение: 1920x1080 или 1280x720
- Битрейт: 2-5 Mbps
- Аудио: AAC, 128-192 kbps

## Конвертация видео

### С помощью FFmpeg
```bash
# Конвертация в MP4 с оптимизацией
ffmpeg -i input.mov -c:v libx264 -preset slow -crf 22 -c:a aac -b:a 128k output.mp4

# Уменьшение размера
ffmpeg -i input.mov -vf scale=1280:720 -c:v libx264 -crf 28 -c:a aac output.mp4
```

### Онлайн конвертеры
- CloudConvert (https://cloudconvert.com)
- Online-Convert (https://www.online-convert.com)
- Convertio (https://convertio.co)

## Проверка

### Тест YouTube трейлера
1. Вставьте YouTube ссылку в админке
2. Откройте страницу фильма
3. Клик на Play → должен открыться YouTube плеер

### Тест локального трейлера
1. Загрузите .mp4 файл через админку
2. Откройте страницу фильма
3. Клик на Play → должен открыться HTML5 плеер с контролами

## Устранение проблем

### Видео не воспроизводится
**Причина:** Неподдерживаемый формат  
**Решение:** Конвертируйте в MP4 (H.264)

### Видео скачивается
**Причина:** Неправильный Content-Type в S3  
**Решение:** Перезагрузите файл через админку

### Видео тормозит
**Причина:** Слишком большой размер  
**Решение:** Уменьшите разрешение или битрейт

### Нет звука
**Причина:** Проблема с аудио кодеком  
**Решение:** Используйте AAC аудио кодек
