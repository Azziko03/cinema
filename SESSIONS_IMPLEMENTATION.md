# Реализация страницы Сеансы

## Что было реализовано

### 1. Страница сеансов (`/admin/sessions`)

Создана полнофункциональная страница для управления сеансами кинотеатра со следующими возможностями:

#### Основные функции:
- **Отображение списка сеансов** с информацией:
  - Постер и название фильма
  - Зал проведения
  - Дата и время начала
  - Базовая цена билета
  - **Детальная статистика занятости мест:**
    - VIP места: прогресс-бар с иконкой звезды (желтый)
    - Обычные места: прогресс-бар с иконкой пользователя (синий)
    - Общая занятость: итоговый прогресс-бар (красный)
    - Числовые показатели для каждой категории (занято/всего)
  
- **Модальное окно добавления сеанса** с возможностью:
  - Выбора фильма из списка с отображением постера, названия, длительности и возрастного рейтинга
  - Изменения статуса фильма прямо из модалки (В прокате / Скоро / Архив)
  - Выбора зала
  - Указания даты и времени начала
  - Установки базовой цены (с защитой от случайного изменения при скролле)
  - Выбора языка (Оригинал / Дубляж)
  - Выбора формата (2D / 3D / IMAX)

- **Редактирование сеансов:**
  - Кнопка "Изменить" открывает модалку с предзаполненными данными
  - Возможность изменить все параметры сеанса
  - Проверка на конфликты расписания при обновлении

- **Удаление сеансов:**
  - Кнопка "Удалить" открывает модалку подтверждения
  - Защита от удаления сеансов с проданными билетами
  - Информативное сообщение о последствиях удаления

#### Валидация:
- Проверка заполнения всех обязательных полей
- Автоматический расчет времени окончания на основе длительности фильма
- Проверка на конфликты расписания (зал не может быть занят в одно время)
- Защита от случайного изменения числовых полей при скролле мышью

### 2. Файлы

#### Клиентский компонент
**Файл:** `app/admin/sessions/SessionsClient.tsx`
- Управление состоянием модальных окон (добавление, редактирование, удаление)
- Обработка форм добавления и редактирования сеанса
- Изменение статуса фильмов
- **Детальное отображение занятости мест:**
  - Отдельные прогресс-бары для VIP и обычных мест
  - Визуальная дифференциация с иконками и цветами
  - Итоговая статистика по всем местам
- Защита input[type="number"] от изменения при скролле через `onWheel` обработчик

#### Серверный компонент
**Файл:** `app/admin/sessions/page.tsx`
- Проверка авторизации администратора
- Загрузка данных сеансов с детальной информацией о местах
- Подсчет занятых VIP и обычных мест для каждого сеанса
- Преобразование типов данных для клиента

#### API Endpoints

**Файл:** `app/api/admin/sessions/route.ts`

**POST /api/admin/sessions** - Создание нового сеанса
- Валидация входных данных
- Проверка существования фильма и зала
- Проверка на конфликты расписания
- Создание записи в базе данных

**DELETE /api/admin/sessions?id={id}** - Удаление сеанса
- Проверка наличия проданных билетов
- Удаление только если нет проданных билетов

**Файл:** `app/api/admin/sessions/[id]/route.ts`

**PUT /api/admin/sessions/{id}** - Обновление сеанса
- Валидация входных данных
- Проверка существования сеанса, фильма и зала
- Проверка на конфликты расписания (исключая текущий сеанс)
- Обновление всех параметров сеанса

**Файл:** `app/api/admin/movies/[id]/route.ts`

**PATCH /api/admin/movies/{id}** - Обновление статуса фильма
- Быстрое изменение только статуса без полного обновления
- Валидация допустимых значений статуса

### 3. Особенности реализации

#### Интеграция с существующей системой:
- Использует существующие компоненты (Modal, ToastContainer)
- Следует паттернам проекта (серверные + клиентские компоненты)
- Интегрирована с Prisma ORM и базой данных PostgreSQL
- Работает аналогично странице фильмов (единый UX)

#### UX/UI улучшения:
- **Детальная визуализация занятости мест:**
  - Три отдельных прогресс-бара (VIP, обычные, общее)
  - Цветовая дифференциация (желтый для VIP, синий для обычных, красный для общего)
  - Иконки для быстрой идентификации типа мест
  - Числовые показатели рядом с каждым прогресс-баром
- Визуальная индикация выбранного фильма в модалке
- Цветовые бейджи для статусов фильмов
- Адаптивный дизайн таблицы
- Состояние загрузки для всех операций
- **Защита от случайного изменения цены при скролле** через `onWheel` обработчик
- Модальные окна для редактирования и удаления (как в странице фильмов)

#### Безопасность:
- Проверка роли администратора на всех эндпоинтах
- Валидация данных на сервере
- Защита от конфликтов расписания

## Как использовать

### Добавление сеанса:
1. Перейдите в админ-панель `/admin/sessions`
2. Нажмите кнопку "Добавить сеанс"
3. В модальном окне:
   - Выберите фильм из списка (можно изменить его статус)
   - Выберите зал
   - Укажите дату и время
   - Установите цену (защищено от случайного изменения при скролле)
   - Выберите язык и формат
4. Нажмите "Добавить сеанс"

### Редактирование сеанса:
1. Найдите нужный сеанс в таблице
2. Нажмите кнопку "Изменить"
3. В модальном окне измените нужные параметры
4. Нажмите "Обновить сеанс"

### Удаление сеанса:
1. Найдите нужный сеанс в таблице
2. Нажмите кнопку "Удалить"
3. Подтвердите удаление в модальном окне
4. Сеанс будет удален (если нет проданных билетов)

### Просмотр статистики мест:
- В колонке "Занято мест" отображается:
  - **VIP места** (желтая звезда): прогресс-бар и числа
  - **Обычные места** (синий человечек): прогресс-бар и числа
  - **Всего**: итоговый прогресс-бар и общее количество

## Реализованные правки

### 1. ✅ Детальное отображение занятых мест
Вместо простого прогресс-бара теперь показывается:
- Отдельная статистика для VIP мест (желтый цвет, иконка звезды)
- Отдельная статистика для обычных мест (синий цвет, иконка пользователя)
- Общая статистика (красный цвет)
- Числовые показатели для каждой категории

### 2. ✅ Защита от изменения при скролле
Добавлен обработчик `onWheel={(e) => e.currentTarget.blur()}` для поля "Базовая цена", который убирает фокус с input при попытке скролла, предотвращая случайное изменение значения.

### 3. ✅ Кнопки изменить и удалить работают как в странице фильмы
- Кнопка "Изменить" открывает модалку с предзаполненными данными
- Кнопка "Удалить" открывает модалку подтверждения с предупреждением
- Единый UX со страницей фильмов
- Все операции с обработкой ошибок и уведомлениями

- **Database:** PostgreSQL + Prisma ORM
