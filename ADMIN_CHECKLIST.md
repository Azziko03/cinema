# ✅ Чеклист настройки админ-панели

Следуйте этому чеклисту для быстрой настройки админ-панели.

## Предварительные требования

- [ ] PostgreSQL установлен и запущен
- [ ] База данных создана
- [ ] Миграции применены (`npm run db:push`)
- [ ] Seed выполнен (`npm run db:seed`)
- [ ] Сервер разработки запущен (`npm run dev`)

## Шаг 1: Telegram бот

- [ ] Открыл [@BotFather](https://t.me/BotFather) в Telegram
- [ ] Отправил команду `/newbot`
- [ ] Придумал имя бота (например: "Cinema Admin Bot")
- [ ] Придумал username бота (например: "cinema_admin_bot")
- [ ] Получил и сохранил токен бота
  ```
  Токен выглядит так: 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
  ```
- [ ] Начал диалог с ботом (отправил `/start`)

## Шаг 2: Telegram User ID

- [ ] Открыл [@userinfobot](https://t.me/userinfobot) в Telegram
- [ ] Отправил любое сообщение боту
- [ ] Получил и сохранил User ID
  ```
  User ID выглядит так: 123456789
  ```

## Шаг 3: Настройка БД

### Вариант A: Через Prisma Studio (рекомендуется)

- [ ] Открыл Prisma Studio: `npm run db:studio`
- [ ] Перешел в таблицу `Setting`
- [ ] Нашел запись с `key = "ADMIN_TELEGRAM_BOT_TOKEN"`
- [ ] Вставил токен бота в поле `value`
- [ ] Нашел запись с `key = "ADMIN_TELEGRAM_USER_ID"`
- [ ] Вставил User ID в поле `value`
- [ ] Сохранил изменения

### Вариант B: Через SQL

- [ ] Подключился к БД
- [ ] Выполнил SQL:
  ```sql
  UPDATE settings 
  SET value = 'ВАШ_ТОКЕН_БОТА' 
  WHERE key = 'ADMIN_TELEGRAM_BOT_TOKEN';

  UPDATE settings 
  SET value = 'ВАШ_USER_ID' 
  WHERE key = 'ADMIN_TELEGRAM_USER_ID';
  ```

## Troubleshooting

### ❌ Код не приходит в Telegram

- [ ] Проверил, что начал диалог с ботом (`/start`)
- [ ] Проверил токен бота в таблице `settings`
- [ ] Проверил User ID в таблице `settings`
- [ ] Проверил логи сервера на наличие ошибок
- [ ] Попробовал отправить тестовое сообщение через Telegram Bot API:
  ```bash
  curl -X POST "https://api.telegram.org/bot<TOKEN>/sendMessage" \
    -d "chat_id=<USER_ID>" \
    -d "text=Test"
  ```

### ❌ Ошибка "Telegram settings not found"

- [ ] Проверил, что записи существуют в таблице `settings`
- [ ] Проверил, что `key` написан правильно (с учетом регистра)
- [ ] Перезапустил сервер разработки

### ❌ Ошибка "Доступ запрещен"

- [ ] Проверил, что пользователь существует
- [ ] Проверил, что `role = "admin"`
- [ ] Проверил, что `status = "active"`
- [ ] Проверил, что `emailVerified = true`

### ❌ Ошибка "Неверный email или пароль"

- [ ] Проверил правильность email
- [ ] Проверил правильность пароля (по умолчанию: `admin123`)
- [ ] Проверил, что `passwordHash` не пустой

### ❌ Перенаправление не работает

- [ ] Очистил кеш браузера
- [ ] Проверил файл `middleware.ts`
- [ ] Перезапустил сервер разработки
- [ ] Проверил консоль браузера на наличие ошибок
